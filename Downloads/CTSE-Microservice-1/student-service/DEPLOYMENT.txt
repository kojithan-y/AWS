Student Service — deployment checklist (SE4010)
================================================

1) Public GitHub repository
   - Push this folder (or the whole group repo). If the repo root is only `student-service`, move workflow + files to root and adjust paths in `.github/workflows/ci.yml`.

2) Environment variables (cloud / ECS / Container Apps / etc.)
   - PORT (default 3000)
   - NODE_ENV=production
   - MONGODB_URI = MongoDB connection string (e.g. mongodb+srv://... or mongodb://host:27017/dbname from Atlas / managed Mongo)
   - EXAM_SERVICE_URL = https://your-exam-service... (no trailing slash)
   - TEACHER_SERVICE_URL = https://your-teacher-service...
   - DEMO_TEACHER_ID = optional teacher id for GET /students/:id/summary demo
   - CORS_ORIGIN = your front-end origin if needed

3) SonarCloud (managed SAST — assignment requirement)
   - Create a free account at https://sonarcloud.io and import the GitHub repo.
   - Generate SONAR_TOKEN and add it as a GitHub Actions secret.
   - Edit `sonar-project.properties`: set `sonar.organization` to your SonarCloud org key and `sonar.projectKey` to a unique key SonarCloud gives you (often org_projectname).

4) Optional: Snyk (also counts as DevSecOps SAST)
   - Install CLI locally: `npm install -g snyk` then `snyk auth` and `snyk test` inside `student-service`.

5) Container registry + orchestration (free tier)
   - Build: `docker build -t your-registry/student-service:tag .`
   - Push to GHCR, ECR, or ACR; deploy to ECS Fargate, Azure Container Apps, or GKE Autopilot trial — keep within provider free tier limits.

6) Security groups / IAM (for your report)
   - Only expose the load balancer / ingress to 80/443; tasks run without public IPs where possible.
   - Use least-privilege IAM roles for the task to pull images and write logs only.

7) Integration demo with teammates
   - Agree on Exam service paths: this service calls GET /health and GET /exams/student/{studentId} on EXAM_SERVICE_URL.
   - Agree on Teacher service: GET /teachers/{id} when DEMO_TEACHER_ID is set.
   - If paths differ, change `src/services/integrationClient.js` to match their OpenAPI.
